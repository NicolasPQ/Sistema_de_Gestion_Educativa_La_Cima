## 📌 Sobre el Proyecto

Sistema web de gestión educativa desarrollado como trabajo académico para la asignatura de **Análisis y Diseño de Sistemas II** en la Universidad de Aquino Bolivia (UDABOL). El sistema centraliza y automatiza los procesos administrativos y académicos de una unidad educativa, reemplazando el manejo manual de registros en papel por una plataforma digital moderna, responsiva y segura.

Abarca la gestión de personal (docentes, administrativos, conserjes), alumnado (estudiantes, tutores), estructura académica (cursos, materias, kárdex), registro de notas y asistencias, control de deudas y sueldos, con autenticación por roles.

> ⚠️ Este repositorio es de carácter público con fines de presentación. El código fuente completo está disponible en el repositorio privado del proyecto.

## 🛠️ Stack Tecnológico

| Categoría     | Tecnología                        |
| ------------- | --------------------------------- |
| Lenguaje      | C#                                |
| Framework Web | ASP.NET Framework MVC             |
| Frontend      | Razor Pages, Bootstrap 5          |
| Base de Datos | Microsoft SQL Server              |
| Autenticación | Sesiones + Roles (RBAC)           |
| Metodología   | Espiral (iterativo e incremental) |
| IDE           | Visual Studio                     |

## 📋 Módulos del Sistema

```
Sistema de Gestión Educativo
├── 🔐 Autenticación          (Roles: Administrador, Secretaría, Docente, Usuario)
├── 📅 Periodos Académicos    (Gestiones anuales, Trimestres)
├── 📚 Académico              (Cursos, Materias, Kárdex)
├── 👤 Usuarios               (Cuentas y roles del sistema)
├── 🏫 Personal               (Administrativos, Docentes, Conserjes)
├── 🎒 Alumnado               (Estudiantes, Tutores)
├── 💳 Finanzas               (Deudas, Facturas, Sueldos)
├── 📝 Notas                  (Por materia, trimestre y campo evaluativo)
└── ✅ Asistencias            (Registro diario por curso y materia)
```

---

## 📐 Metodología

El desarrollo siguió la **metodología en espiral**, con fases iterativas de análisis, diseño, implementación y evaluación. Esto permitió adaptar los requisitos de manera continua e incorporar mejoras en cada ciclo.
